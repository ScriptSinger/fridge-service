В Laravel часто используют Illuminate Pipeline для фильтров и сортировок, особенно когда параметров становится много.

Это называется Query Filter Pipeline.

Идея:

каждый фильтр — отдельный класс

pipeline прогоняет query через фильтры

контроллер остаётся очень тонким

Как это выглядит архитектурно
app
 ├ Filters
 │   ├ BrandFilter.php
 │   ├ DeviceFilter.php
 │   └ SortFilter.php
 │
 ├ Pipelines
 │   └ GalleryPipeline.php
 │
 └ Controllers
     └ GalleryController.php
1. Filter класс (пример)
BrandFilter
namespace App\Filters;

class BrandFilter
{
    public function handle($query, $next)
    {
        $brand = request()->query('brand');

        if ($brand && $brand !== 'all') {
            $query->whereHas('brand', function ($q) use ($brand) {
                $q->whereRaw('LOWER(name) = ?', [strtolower($brand)]);
            });
        }

        return $next($query);
    }
}
DeviceFilter
class DeviceFilter
{
    public function handle($query, $next)
    {
        $device = request()->query('device');

        if ($device && $device !== 'all') {
            $query->whereHas('device', function ($q) use ($device) {
                $q->whereRaw('LOWER(type) = ?', [strtolower($device)]);
            });
        }

        return $next($query);
    }
}
SortFilter
class SortFilter
{
    public function handle($query, $next)
    {
        $sort = request()->query('sort', 'newest');

        $query = match ($sort) {
            'oldest' => $query
                ->orderByRaw('COALESCE(published_at, created_at) ASC')
                ->orderBy('created_at'),

            default => $query
                ->orderByRaw('COALESCE(published_at, created_at) DESC')
                ->orderByDesc('created_at'),
        };

        return $next($query);
    }
}
2. Pipeline
namespace App\Pipelines;

use Illuminate\Pipeline\Pipeline;

class GalleryPipeline
{
    public static function apply($query)
    {
        return app(Pipeline::class)
            ->send($query)
            ->through([
                \App\Filters\BrandFilter::class,
                \App\Filters\DeviceFilter::class,
                \App\Filters\SortFilter::class,
            ])
            ->thenReturn();
    }
}
3. Контроллер становится очень чистым
public function index(Request $request)
{
    $query = Gallery::query()
        ->with(['device','brand','service'])
        ->whereNotNull('image')
        ->where('image','!=','');

    $galleries = GalleryPipeline::apply($query)
        ->paginate(24)
        ->withQueryString();

    return view('pages.gallery', [
        'galleries' => $galleries,
    ]);
}
Почему это считается senior подходом

Pipeline даёт:

1️⃣ Single Responsibility

Каждый класс делает одну вещь.

BrandFilter
DeviceFilter
SortFilter
DateFilter
SearchFilter
2️⃣ Легко добавлять новые фильтры

Добавил класс:

PriceFilter

и просто добавил в pipeline:

->through([
    BrandFilter::class,
    DeviceFilter::class,
    PriceFilter::class,
])
3️⃣ Очень удобно тестировать

Тестируешь каждый фильтр отдельно.

4️⃣ Контроллер остаётся тонким

Это важная практика Laravel.

Когда Pipeline реально стоит использовать

Когда есть:

фильтры

сортировки

search

pagination

сложные query параметры

Например:

?brand=apple
&device=laptop
&sort=newest
&price_min=100
&price_max=500
&search=macbook
